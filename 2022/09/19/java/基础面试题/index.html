集合

Java的集合源自两个接口

Collection和Map

抽象类

  1. 抽象类和接口的共同点和区别

    1. 都不能被实例化
    2. 都可以包括抽象方法
    3. 都可以由默认实现的方法

  2. 子类必须重写父类的抽象方法

List

  1. ArrayList(重点)
    1. 扩容机制
      1. 三种构造方式
        1. 空的构造函数
        2. 传入一个指定的容量
        3. 用一个集合初始化
      2. 一般扩容为当前容量的1.5倍
      3. 需要比较新容量和默认最小容量(10)的大小，取较大值
      4. 插入元素前保证容量够
    2. 线程不安全
  2. Vector
  3. LinkedList

Set

  1. HashSet
  2. LinkedHashSet
  3. TreeSet

Queue

  1. PriorityQueue
  2. ArrayQueue

Map

  1. HashMap(重点)
    1. 链表散列
    2. hashcode + 扰动函数得到hash值 (key == null) ? 0 : (h = key.hashCode()) ^ (h >>> 16);
    3. (n-1) & hash得到位置
    4. 拉链法解决冲突
      1. 数组长度大于64且链表长度超过8时，链表会转为红黑树
      2. 当链表长度超过8但数据长度小于64时，会进行扩容而不是转为红黑树
    5. 数组的长度是2的幂次方，方便运算
    6. 线程不安全，使用ConcurentHashMap
    7. 数组初始大小是16，装载因子默认是0.75(分布的概率)，实时装载因子的计算是所有键值对的个数/数组大小
    8. 扩容时需要重新hash
  2. LinkedHashMap
  3. Hashtable
  4. TreeMap
  5. ConcurrentHashMap
    1. Node数组+链表+红黑树实现
    2. 对Node节点加锁，实现线程安全

Synchronized

  1. 应用范围

    1. 修饰实例方法，获得对象锁
    2. 修饰静态方法，获得类的锁
    3. 修饰代码块，获得类或者对象锁

  2. 实现原理

    1. 同步语句块
      1. 对象监视器monitor的获取
      2. monitorenter、monitorexit
    2. 同步方法
      1. ACC_SYNCHRONIZED标识
      2. 本质上都是对象监视器monitor的获取

  3. 优化

  4. sysnchronized和ReentrantLock的区别

类加载器详解

  1. 类加载的类别

    1. BoostrapClassLoader

    3. AppClassLoader

  2. 类的加载过程

    1. 根据类名找到字节码
    2. 根据字节码生成数据结构并放到方法区
    3. 在内存中生成一个Class对象，作为方法区的数据的访问入口
    4. 重写loadClass方法控制字节流的获取方式
